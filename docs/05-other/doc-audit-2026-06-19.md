### 2.2 ADR numbering gaps with no register

Present: 0001, 0002, 0007–0010, 0012–0017, 0021–0039. **Missing: 0003–0006, 0011,
0018–0020.** Nothing marks these as reserved, abandoned, or superseded. For an
"immutable chronological log" this is a smell — a reader can't tell whether an ADR
was lost or never existed. Add a short "Reserved / withdrawn numbers" note to
`docs/adr/` (or a one-line stub ADR per gap) so the sequence is self-explaining.

### 2.3 Two files named `security.md`

- `docs/04-development/security/security.md` (15.6 KB — the security baseline, referenced by CLAUDE.md §6)
- `docs/04-development/security/runbooks/security.md` (4.9 KB — an operational runbook)

Same filename, different purpose, adjacent directories. This invites wrong-file
edits and ambiguous links. Rename the runbook to something like
`security-incident.md` or `security-runbook.md`.

### 2.4 References to documents that were never written

Beyond the broken links, several docs promise companion files that don't exist yet:

`voice-guide-ru.md` (per ADR-0029, from `speaker-outreach.md`), the `pii-data-flow.md`

## 4. Optimization & enhancement opportunities

These go beyond fixing what's broken.

1. **Add a CI link-check gate.** A broken-link rate of 4.6% will only grow. A
   markdown link linter (e.g. `lychee`, or the script in §5) run in the
   supply-chain workflow would catch regressions at PR time, consistent with the
   "zero warnings" posture in CLAUDE.md §2.10.

2. **Stop linking into agent memory from committed docs.** The 27 `-home-drukker-…`
   links are a systemic pattern: canonical docs were authored citing a local agent's
   private memory. That path is unportable and breaks for every other reader/agent.
   Establish a rule — if a fact matters enough to cite, it belongs in `docs/`, not in
   agent memory. This is the single highest-leverage fix.

3. **Generate the layer indexes from the tree.** ADR-0039 / the migration script
   already generate `README.md` per layer "so they can't drift." Keep that generator
   (extracted from `_restructure/migrate.py` before deletion) as a maintained
   `scripts/` tool rather than throwing it away with the one-shot folder.

4. **ADR index / status table.** 39 ADRs with no single status view. A generated
   `docs/adr/README.md` table (number, title, status, supersedes) would make the log
   navigable and surface the numbering gaps automatically.

5. **A docs ownership / freshness convention.** With ~120 docs and one human
   contributor, a lightweight front-matter convention (`owner`, `last-reviewed`,
   `status: living|frozen|archived`) would let you mechanically flag stale docs —
   especially useful given the project is paused (per `handover.md`) and the next
   reader needs to know what to trust.

## 5. Re-runnable link checker

Drop this in `scripts/check-doc-links.py` and wire into CI:

```python
import os, re, sys
root = "docs"
md = [os.path.join(dp, f) for dp, _, fn in os.walk(root)
      for f in fn if f.endswith(".md")]
link = re.compile(r'\[[^\]]*\]\(([^)]+)\)')
broken = []
for mf in md:
    txt = re.sub(r'```.*?```', '', open(mf, encoding="utf-8", errors="replace").read(), flags=re.S)
    for m in link.finditer(txt):
        t = m.group(1).strip()
        if t.startswith(("http", "mailto:", "#", "tel:")):
            continue
        p = t.split("#")[0]
        if p and not os.path.exists(os.path.normpath(os.path.join(os.path.dirname(mf), p))):
            broken.append((mf, t))
for mf, t in sorted(broken):
    print(f"BROKEN  {mf} -> {t}")
print(f"\n{len(broken)} broken / {len(md)} files", file=sys.stderr)
sys.exit(1 if broken else 0)
```
