Resolves ISS-TEST-WEB-001.

## What

Bumps vitest 2.1.9 to 4.1.9 in apps/web, apps/web-next, and apps/api so the test runner can parse and execute tests that import a sibling module. Companion edits remove an obsolete config workaround and wire @vitejs/plugin-react for .tsx parsing under vite 8.1.0 / rolldown.

## Why

`vitest ^2.1.8` bundles an older vite whose SSR transform is missing `__vite_ssr_exportName__` (that helper was added in vite v8). The workspace's hoisted `vite@8.1.0` defines the helper, so any cross-module `import` in a test crashed the suite load with `ReferenceError: __vite_ssr_exportName__ is not defined`. This blocks ISS-UAT-013-13 AC-3 (a regression test added by wf-20260703-fix-065-onboarding-copy that could not run) and any future test that imports a sibling helper.

## How

- `apps/{web,web-next,api}/package.json` — bump vitest from `^2.1.8` to `^4.1.9`. vitest 4.1.9 declares peer `vite: ^6.0.0 || ^7.0.0 || ^8.0.0`, satisfied by workspace's vite 8.1.0.
- `apps/api/package.json` — bump @vitest/coverage-v8 from `^2.1.8` to `^4.1.9` to match the new vitest series.
- `apps/api/vitest.unit.config.ts` — remove the obsolete `transformMode: 'web'` workaround (option removed in vitest 3.0).
- `apps/web-next/vitest.config.ts` + `apps/web-next/package.json` — wire @vitejs/plugin-react (5.2.0, with `jsxRuntime: 'automatic'`) as the first plugin, mirroring apps/storybook/.storybook/main.ts (PR #109 / ISS-CI-OVERRIDE-ebd184b). Without this, FilterChip.test.tsx fails to parse under vite 8.1.0/rolldown.
- `apps/web/src/lib/utm.test.ts` — replace the 2-line workaround comment with a 9-line historical note linking to ISS-TEST-WEB-001 (no test logic change).
- `pnpm-lock.yaml` — auto-regenerated by `pnpm install --no-frozen-lockfile`.
