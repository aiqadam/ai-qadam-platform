### 5. `apps/web-next/vitest.config.ts` — wire `@vitejs/plugin-react`

Same root-cause as PR #109 (ISS-CI-OVERRIDE-ebd184b) — vite 8.1.0 / rolldown
parses `.tsx` with JSX disabled by default. The repo's adopted pattern is
to install `@vitejs/plugin-react` and prepend `react({ jsxRuntime:
'automatic' })` as the FIRST plugin in the chain so Babel transpiles JSX
before rolldown's parser sees it. Without this, all 33 test files in
`apps/web-next` would still pass (32 .ts files) EXCEPT for
`src/blocks/workspace/FilterChip.test.tsx` (a `.tsx` test), which now passes
13 assertions across 9 tests.

## Why vitest 4.1.9 not 3.x

vitest 4.1.9 declares peer `vite: ^6.0.0 || ^7.0.0 || ^8.0.0`. The
workspace's hoisted vite 8.1.0 (transitive via `@tailwindcss/vite` and
`@astrojs/react`) satisfies exactly. Bumping to 4.x keeps us on the
current major that vitest itself supports going forward and avoids a
future re-bump when vitest 2.x (or 3.x) reaches end-of-life.
