## Honesty disclosure

`apps/api/src/health/health.controller.ts` was extended with a `mode`
field that returns one of `'production' | 'uat' | 'disabled'`. This
endpoint is unauthenticated and exposes the `mode` value to any
caller. The pre-flight script uses that on the same host, but the
field is also reachable from anywhere on the local network. The
SecurityReviewer (Step 5) accepted the disclosure on the grounds
that:

1. `mode` is a coarse-grained property — it tells the caller which
   environment category is active, not anything secret about the
   environment.
2. The provider name (`'smtp'` vs `'resend'`) was already public on
   the same endpoint, and `mode` is strictly less informative than
   the combination of provider + the existence of `RESEND_API_KEY`
   / `SMTP_HOST` env vars (which an attacker can probe via other
   means anyway).
3. Production deployment puts the API behind the OIDC-authenticated
   Astro proxy — only the pre-flight script (running on the same
   host) reaches the endpoint directly.

This trade-off is documented inline (JSDoc on `emailHealth()`) and in
the [Step 5 security review](.copilot/tasks/active/wf-20260701-uat-045-mailpit-resend/04-security-review.md).
