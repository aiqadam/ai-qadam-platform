# Step 0.5 — Context Sync (FEAT-WORKFLOW-001)

**Status:** passed (after corrective fix)
**Date:** 2026-07-04

## Initial run

```
$ bash scripts/check-workflow-state.sh --base "origin/main"
DRIFT: issues/registry.md references 'ISS-CI-OVERRIDE-'
ERROR: 1 drift item(s) detected against origin/main.
```

## Root cause analysis

The drift detector's `extract_issue_ids()` regex `ISS-[A-Z0-9-]+` did not
include lowercase hex characters. AGENTS.md §6.3 step 3 names the
PRSteward auto-registration pattern as `ISS-CI-OVERRIDE-<sha1-prefix>`.
PR #94 auto-registered `ISS-CI-OVERRIDE-ebd184b` (sha1 prefix of the
rolldown PARSE_ERROR class). The greedy regex matched `ISS-CI-OVERRIDE-`
(the trailing `-` IS in the character class) but stopped at the lowercase
hex `e`, producing a phantom orphan ID with no corresponding file.

This is a **false positive in the drift detector** — the actual file
`.copilot/issues/ISS-CI-OVERRIDE-ebd184b.md` exists and is referenced
correctly from the registry. The drift gate was blocking every future
workflow from starting.

## Corrective fix

Updated `extract_issue_ids()` to use `ISS-[A-Z0-9a-f-]+` and added a
comment block citing the AGENTS.md §6.3 naming convention so future
maintainers understand why lowercase hex is required.

Also added a regression test in `scripts/tests/check-workflow-state.bats`
that:

1. Creates `.copilot/issues/ISS-CI-OVERRIDE-ebd184b.md` and a registry
   row referencing it.
2. Asserts the drift detector exits 0 (no phantom drift).
3. Asserts the diagnostic does not contain `ISS-CI-OVERRIDE-' ` or
   `ISS-CI-OVERRIDE-.` substrings (the truncated-prefix patterns the
   broken regex produced).
