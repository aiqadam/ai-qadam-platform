## Changes

### 1. `apps/storybook/package.json`

Added a direct `devDependencies` entry for `@vitejs/plugin-react@^5.2.0`.
The package was already in the workspace tree (resolved transitively via
`@astrojs/react` → `@vitejs/plugin-react`), but declaring it as a direct
devDep documents the intent, ensures pnpm hoisting preserves it through
future dependency tree changes, and matches the documented Storybook 8
+ Vite + React integration pattern.

### 2. `apps/storybook/.storybook/main.ts`

- Added `import react from '@vitejs/plugin-react';` at the top.
- Updated the file-level comment to explain why this plugin is required
  (rolldown 1.1.3 parses `.tsx` without JSX; this plugin transpiles JSX
  away before the bundler runs).
- Updated `viteFinal` to prepend `react({ jsxRuntime: 'automatic' })`
  **before** all other plugins in the chain. `react()` is registered
  without an explicit `enforce` field because storybook's `viteFinal`
  runs before Storybook composes its own plugins, so the
  prepend-position alone is sufficient. (`'pre'` is the default for
  React's own internal ordering.)
- `tailwindcss()` plugin stays after storybook's existing plugins to
  preserve current behavior.

### 3. `pnpm-lock.yaml`

Will be regenerated by `pnpm install --lockfile-only` to record the
direct devDep declaration. The actual on-disk resolution is unchanged
because the package was already present in the tree.
